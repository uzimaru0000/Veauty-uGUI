# Widget Catalog

[日本語](ja/widgets.md)

Every widget class in `Veauty.uGUI`, the `GUIBase<T>` lifecycle, and `Ref<T>`.

## `GUIBase<T>` — the widget base class

```csharp
public abstract class GUIBase<T> :
    Node<UnityEngine.GameObject, T>,
    IHostLifecycle<UnityEngine.GameObject>,
    IHostLifecycleTree<UnityEngine.GameObject>
    where T : UnityEngine.Component
```

A widget is a VTree node whose tag is `typeof(T).FullName` and whose rendered GameObject carries component `T`. Constructor: `(IEnumerable<IAttribute<GameObject>> attrs, params IVTree[] kids)`.

### Lifecycle hooks

| Hook | When | Typical use |
|---|---|---|
| `Init(go)` | after GameObject + component creation, **before** attributes and children | build internal children (Slider fill/handle, InputField text area, Dropdown template) |
| `AfterRenderKids(go)` | after all VTree children are rendered and parented | wiring that needs children (ScrollRect `content`) |
| `Destroy(go)` | before unmount | cleanup (default: no-op) |

Full ordering diagram in [Architecture](architecture.md). Internal children created in `Init` are skipped by the patcher via the child-offset rule — never create internal children outside `Init`.

### Protected helpers (for custom widgets)

| Helper | Description |
|---|---|
| `CreateChild(parent, name)` | new GameObject + `RectTransform`, parented with `worldPositionStays: false` |
| `Stretch(rt)` | anchors 0..1, `sizeDelta = 0` |
| `AddVisual<TComp>(go, color)` | adds `CanvasRenderer` + graphic `TComp`, sets color |
| `SetupRect(rt, ...)` | sets only the non-null RectTransform properties |
| `FindPart<T>()` | first kid of type `T` from the raw kids (used to read sub-components) |

## Hand-written widgets

These are not produced by the generator (they are in the generator's `ManualTypes` skip list):

| Widget | Component | Attributes |
|---|---|---|
| `Canvas` | `UnityEngine.Canvas` | `RenderMode`, `WorldCamera`, `PlaneDistance`, `PixelPerfect`, `ScaleFactor`, `ReferencePixelsPerUnit`, `OverridePixelPerfect`, `OverrideSorting`, `SortingOrder`, `TargetDisplay`, `SortingLayerID`, `SortingLayerName`, `AdditionalShaderChannels`, `NormalizedSortingGridSize`, `UseReflectionProbes`, `VertexColorAlwaysGammaSpace`, `UpdateRectTransformForStandalone` |
| `CanvasGroup` | `UnityEngine.CanvasGroup` | `Alpha`, `Interactable`, `BlocksRaycasts`, `IgnoreParentGroups` |
| `RectTransform` | `UnityEngine.RectTransform` | `AnchorMin`, `AnchorMax`, `AnchoredPosition`, `AnchoredPosition3D`, `SizeDelta`, `Pivot`, `OffsetMin`, `OffsetMax`, `SendChildDimensionsChange` |

`RectTransform` attribute classes are usually passed to *other* widgets (every uGUI node has a RectTransform):

```csharp
V.Image(extras: new IAttribute<GameObject>[] {
    new RectTransform.AnchorMin(Vector2.zero),
    new RectTransform.AnchorMax(Vector2.one),
})
```

Rendering a `Canvas` widget does **not** add a `GraphicRaycaster` or create an `EventSystem`.

## Generated widgets (`Lib/Generated/`)

One widget per public `UnityEngine.UI` MonoBehaviour subclass, regenerated by **Veauty > GenerateUIClass** ([Code generation](code-generation.md)). Current set (31 types):

`AspectRatioFitter`, `BaseMeshEffect`, `Button`, `CanvasScaler`, `ContentSizeFitter`, `Dropdown`, `Graphic`, `GraphicRaycaster`, `GridLayoutGroup`, `HorizontalLayoutGroup`, `HorizontalOrVerticalLayoutGroup`, `Image`, `InputField`, `LayoutElement`, `LayoutGroup`, `Mask`, `MaskableGraphic`, `Outline`, `PositionAsUV1`, `RawImage`, `RaycastReceiver`, `RectMask2D`, `Scrollbar`, `ScrollRect`, `Selectable`, `Shadow`, `Slider`, `Text`, `Toggle`, `ToggleGroup`, `VerticalLayoutGroup`

Notes:

- Attribute classes are generated **per declared property only** — `Button` has just `OnClick`; to set `interactable` on a Button use `Selectable.Interactable` (base-class attributes work on derived components because `GetComponent<Selectable>()` finds the Button).
- `HorizontalLayoutGroup` / `VerticalLayoutGroup` / `GridLayoutGroup` declare no own writable properties; their usable attributes live in [LayoutAliases](#layout-aliases) or on `HorizontalOrVerticalLayoutGroup` / `LayoutGroup`.
- Abstract-component widgets (`Graphic`, `MaskableGraphic`, `LayoutGroup`, `Selectable`, `HorizontalOrVerticalLayoutGroup`, `BaseMeshEffect`) exist mainly as attribute containers; instantiating them as widgets would try to `AddComponent` an abstract type and fail — use their attributes on concrete widgets instead.
- Full attribute tables per widget: [API reference](api-reference.md).

## `Ref<T>` — capturing rendered GameObjects

`Ref<T>` is an attribute that stores the object it is applied to instead of mutating it:

```csharp
var panelRef = new Ref<GameObject>();          // global-namespace alias

IVTree Render() =>
    V.Image(color: Color.black, extras: new IAttribute<GameObject>[] { panelRef })[
        ...
    ];

// after render: panelRef.current is the Image's GameObject
var image = panelRef.current.GetComponent<UnityEngine.UI.Image>();
```

- `current` holds the captured object (constructor default before first render).
- The attribute key is `"ref:" + GetHashCode()`, unique per instance.
- There are **two** `Ref<T>` types: `Veauty.uGUI.Ref<T>` and a global-namespace `Ref<T>` subclass provided so user code doesn't need a `using`. They are interchangeable; prefer the namespaced one in library code.

## `ImageStyle`

A small value struct (`Sprite`, `Color`, `ImageType`) with `ApplyTo(Image)` — sprite applied only when non-null, color and type always. Available for custom widget code.
